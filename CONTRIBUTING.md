# Contribuer

Merci. Ici, on préfère la clarté à la vitesse.

## Licence des contributions

En soumettant une contribution (code, documentation, exemples, tests),
vous acceptez que votre contribution soit publiée sous la **GNU GPL v3 (GPL-3.0-or-later)**,
comme le reste du dépôt.

Si vous ne pouvez pas accepter cela, n’ouvrez pas de PR : ouvrez plutôt une issue.

## Ce qu’on attend d’une PR

- Une description simple : *qu’est-ce qui change, et pourquoi ?*
- Des tests quand c’est pertinent.
- Un style lisible (même si le projet est “symbolique”, le code doit rester vérifiable).

## Ce qu’on refuse

- Code copié depuis une source dont la licence est incompatible avec la GPL.
- Données/poids/modèles ajoutés sans provenance claire.
- Ajouts qui rendent le cœur du système opaque “par design”.

## Signalement de problèmes

Ouvrez une issue avec :
- étapes de reproduction
- sortie console / log
- version Python / OS
